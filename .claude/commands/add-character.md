新キャラクターをゲームに追加してください。以下の手順を順番に実行します。

## 手順

### 1. キャラクター情報のヒアリング

ユーザに以下の情報を確認してください:

- **名前**: キャラクター名（日本語）
- **HP**: 体力（既存キャラの範囲: 80〜150）
- **攻撃力**: attack値（既存キャラの範囲: 20〜50）
- **防御力**: defense値（既存キャラの範囲: 5〜35）
- **コンセプト**: どんな特徴のキャラか（例: 高火力・低耐久のアタッカー）

既存キャラクターの一覧を参考に提示してください:

```
ドラゴン:   HP=105, ATK=30, DEF=15（バランス型）
ナイト:     HP=100, ATK=25, DEF=30（防御型）
ウィザード: HP=80,  ATK=45, DEF=10（攻撃型）
ゴーレム:   HP=150, ATK=20, DEF=35（高耐久型）
アサシン:   HP=80,  ATK=50, DEF=5 （超攻撃型）
プリースト: HP=90,  ATK=25, DEF=25（回復型）
```

### 2. `CHARACTERS` 配列に追加

`packages/shared/src/characters.ts` の `CHARACTERS` 配列末尾に新キャラクターを追加してください。

```typescript
// 例:
{ name: "新キャラ名", hp: XX, attack: XX, defense: XX },
```

**注意**: 配列のインデックス（1-based）がキャラクターIDになります。モバイルアプリのガチャ機能がこのインデックスを使用するため、既存エントリの順序は変更しないでください。

### 3. seed.ts の自動参照を確認

`apps/server/prisma/seed.ts` は `CHARACTERS` 配列を自動参照してDBに同期するため、変更不要です。念のためファイルを確認し、`CHARACTERS` をimportしていることを確認してください。

### 4. SQLiteに同期

```
pnpm db:seed
```

管理画面のDBに新キャラクターを反映します。

### 5. シミュレータでバランス検証

```
pnpm dev:server
```

サーバを起動し、ブラウザで http://localhost:3000/admin/simulator にアクセスしてください。

確認ポイント:
- 新キャラクターがシミュレータのキャラ選択に表示されるか
- 全マッチアップの勝率を確認（目標: 各キャラの総合勝率が **40〜60%** の範囲）
- 極端に有利/不利なマッチアップがないか
- 平均ターン数が妥当か（膠着しすぎていないか）

結果をユーザに報告してください。

### 6. ステータス調整（必要に応じて）

シミュレータの結果が偏っている場合:
1. ステータスを調整（`packages/shared/src/characters.ts`）
2. `pnpm db:seed` で再同期
3. シミュレータで再検証

ユーザが納得するまで繰り返してください。

### 7. キャラクター画像の確認

キャラクター画像は `/characters/{id}/idle.png` の規約に従います（idは1-based）。新キャラのIDを確認し、画像が必要であることをユーザに伝えてください。

### 8. ドキュメント更新

以下のドキュメントを更新してください:

- `README.md` — キャラクター一覧テーブルに新キャラを追記
- `CLAUDE.md` — キャラクター数やキャラ関連の記述があれば更新

### 9. 完了レポート

以下の情報をまとめて報告してください:

- 追加キャラクター: 名前 / HP / ATK / DEF
- キャラクターID: （CHARACTERS配列でのインデックス、1-based）
- シミュレータ検証結果: 総合勝率
- ドキュメント更新: 済 / 未
- 画像: 配置先パス（`/characters/{id}/idle.png`）
- アプリ反映: 実機ビルド（`/dev-wifi`）で反映可能
