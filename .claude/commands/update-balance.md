バトルパラメータを調整してください。以下の手順を順番に実行します。

## 手順

### 1. 変更意図のヒアリング

ユーザに以下を確認してください:

- **何を変えたいか**: どのパラメータ、またはどのキャラクターのステータスか
- **なぜ変えたいか**: 現在の問題点（例: 特定キャラが強すぎる、バトルが長すぎる）

現在のパラメータを参考に提示してください:

**バトル定数** (`packages/shared/src/constants.ts`):
```
ターン制限時間:       15秒
最低ダメージ:         1
防御倍率:             1.5x
ダメージ変動:         ±15%
必殺技クールダウン:   3ターン
必殺技倍率:           1.8x
カウンター成功率:     30%
カウンターダメージ:   1.5x
```

**経験値・レベル定数**:
```
最大レベル:           Lv20
勝利EXP:              30
敗北EXP:              10
最低EXP:              5
最大EXP:              50
レベル補正:           +2%/Lv
```

**キャラクターステータス** (`packages/shared/src/characters.ts`):
```
ドラゴン:   HP=105, ATK=30, DEF=15
ナイト:     HP=100, ATK=25, DEF=30
ウィザード: HP=80,  ATK=45, DEF=10
ゴーレム:   HP=150, ATK=20, DEF=35
アサシン:   HP=80,  ATK=50, DEF=5
プリースト: HP=90,  ATK=25, DEF=25
```

### 2. 対象ファイルの特定と変更

変更対象に応じて適切なファイルを編集してください:

| 変更内容 | 対象ファイル |
|---------|-------------|
| バトル定数（倍率・クールダウン等） | `packages/shared/src/constants.ts` |
| キャラクターステータス（HP・ATK・DEF） | `packages/shared/src/characters.ts` |
| ダメージ計算ロジック | `packages/shared/src/damage.ts` |
| ターン処理ロジック | `packages/shared/src/engine.ts` |
| レベル・経験値計算 | `packages/shared/src/level.ts` |

### 3. SQLite同期（キャラ変更時のみ）

キャラクターステータスを変更した場合のみ実行してください:

```
pnpm db:seed
```

バトル定数のみの変更であれば不要です（定数はコードから直接参照されるため）。

### 4. シミュレータで検証

```
pnpm dev:server
```

サーバを起動し、ブラウザで http://localhost:3000/admin/simulator にアクセスしてください。

**確認ポイント**:
- 全キャラクターの総合勝率が **40〜60%** の範囲か
- 極端に有利/不利なマッチアップ（勝率20%以下 or 80%以上）がないか
- 平均ターン数が妥当か（膠着マッチアップがないか）
- 変更意図通りの効果が出ているか

### 5. 検証結果の報告

シミュレータの結果をユーザに報告してください:

- 変更前後の比較（可能であれば）
- 各キャラクターの総合勝率
- 問題のあるマッチアップ
- さらなる調整の要否

### 6. 追加調整（必要に応じて）

ユーザのフィードバックに基づき:
1. パラメータを再調整
2. キャラ変更があれば `pnpm db:seed`
3. シミュレータで再検証

ユーザが納得するまで繰り返してください。

### 7. ドキュメント更新

変更内容に応じて以下を更新してください:

- `README.md` — キャラクターステータス一覧、バトルシステムの説明
- `CLAUDE.md` — バトルパラメータ値の記載（変更があった場合）

### 8. 完了レポート

以下の情報をまとめて報告してください:

- 変更内容: 変更したパラメータと変更前→変更後の値
- 変更ファイル: 編集したファイル一覧
- シミュレータ検証: 全キャラの総合勝率サマリ
- ドキュメント更新: 済 / 未
- アプリ反映: 実機ビルド（`/dev-wifi`）で反映可能
